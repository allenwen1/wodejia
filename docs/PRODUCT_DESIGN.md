# 我爱我家系统 - 点菜系统设计文档

## 一、产品概述

### 1.1 产品定位
"我爱我家系统"是一款面向家庭场景的点菜管理系统，核心功能是让家庭成员（特别是女主人）方便地记录、管理和分享菜品信息，解决"今天吃什么"的日常难题。

### 1.2 目标用户
- **主要用户**：家庭主妇/主夫（菜品管理者）
- **次要用户**：家庭成员（浏览、点菜、评价）

### 1.3 核心价值
- 📸 **可视化菜品库**：用图片记录每道菜，一目了然
- 📝 **口味档案**：记录家人的口味偏好，做菜更贴心
- 🍽️ **智能点菜**：根据人数、口味快速规划菜单
- 💝 **爱的表达**：通过菜品记录传递对家人的关心

## 二、功能模块设计

### 2.1 菜品管理模块

#### 2.1.1 菜品录入
| 字段 | 说明 | 必填 |
|------|------|------|
| 菜品图片 | 支持多张图片上传，展示成品效果 | ✅ |
| 菜品名称 | 如：红烧肉、番茄炒蛋 | ✅ |
| 菜品分类 | 荤菜/素菜/汤羹/主食/小吃/饮品 | ✅ |
| 口味标签 | 酸甜/麻辣/清淡/咸鲜/香辣/酱香等 | ❌ |
| 难度等级 | ⭐简单 / ⭐⭐中等 / ⭐⭐⭐困难 | ❌ |
| 烹饪时间 | 预计制作时长（分钟） | ❌ |
| 适合人数 | 1-2人 / 3-4人 / 5人以上 | ❌ |
| 食材清单 | 主料、辅料列表 | ❌ |
| 制作步骤 | 文字描述或视频链接 | ❌ |
| 小贴士 | 烹饪技巧、注意事项 | ❌ |

#### 2.1.2 菜品浏览
- 网格/列表双视图
- 按分类筛选
- 按口味筛选
- 按难度筛选
- 搜索功能（名称、食材）
- 收藏/常用标记

#### 2.1.3 菜品详情
- 大图轮播展示
- 完整信息展示
- 编辑/删除功能
- 分享功能（生成图片卡片）

---

### 2.2 口味偏好模块

#### 2.2.1 家庭成员管理
- 添加家庭成员（头像、昵称、角色）
- 设置每个人的口味档案

#### 2.2.2 口味档案
| 维度 | 选项 |
|------|------|
| 辣度偏好 | 不吃辣 / 微辣 / 中辣 / 重辣 |
| 甜度偏好 | 无糖 / 少糖 / 正常 / 喜甜 |
| 咸淡偏好 | 清淡 / 适中 / 偏咸 |
| 忌口食材 | 海鲜 / 牛羊肉 / 香菜 / 葱姜蒜等 |
| 最爱菜品 | 标记最喜欢的几道菜 |
| 讨厌菜品 | 标记不吃的菜品 |

#### 2.2.3 智能推荐
- 根据用餐成员自动过滤忌口菜品
- 优先推荐符合多数人口味的菜品
- 标记"全家都爱"的招牌菜

---

### 2.3 点菜系统模块（核心功能）

#### 2.3.1 创建菜单
- **用餐场景**：日常晚餐 / 周末聚餐 / 节日宴请 / 早餐 / 夜宵
- **用餐人数**：滑动选择 1-10+ 人
- **选择成员**：勾选本次用餐的家庭成员
- **用餐日期**：今天 / 明天 / 指定日期

#### 2.3.2 智能配菜
- **一键生成**：根据人数和口味偏好自动推荐菜单
- **荤素搭配**：自动平衡荤素比例
- **菜品数量建议**：N人用餐推荐 N+1 或 N+2 道菜
- **重复提醒**：近期做过的菜给出提示

#### 2.3.3 手动点菜
- 从菜品库浏览选择
- 按分类快速挑选
- 实时显示已选菜品列表
- 显示预计总用时
- 显示食材汇总清单（方便采购）

#### 2.3.4 菜单管理
- 保存历史菜单
- 重复使用该菜单
- 菜单评价反馈
- 生成购物清单

---

### 2.4 购物清单模块

#### 2.4.1 自动生成
根据选定的菜单，自动汇总所需食材，生成购物清单。

#### 2.4.2 清单管理
- 勾选已购买食材
- 添加额外采购项
- 按食材分类（蔬菜/肉类/调料等）
- 分享清单（微信/短信）

---

### 2.5 数据统计模块

#### 2.5.1 菜品统计
- 最常做的菜 TOP10
- 各分类菜品数量
- 新增菜品趋势

#### 2.5.2 用餐统计
- 本周/本月做饭次数
- 最受欢迎菜单
- 平均用餐人数

#### 2.5.3 口味分析
- 家庭成员口味偏好分布
- 荤素比例分析

## 四、技术实现建议

### 4.1 数据库设计

#### 菜品表 (dishes)
```sql
- id: 主键
- name: 菜品名称
- category_id: 分类ID
- images: 图片URL数组
- taste_tags: 口味标签数组
- difficulty: 难度等级
- cook_time: 烹饪时间
- serving_size: 适合人数
- ingredients: 食材清单
- steps: 制作步骤
- tips: 小贴士
- is_favorite: 是否收藏
- created_by: 创建用户ID
```

#### 家庭成员表 (family_members)
```sql
- id: 主键
- name: 昵称
- avatar: 头像URL
- role: 角色（妈妈/爸爸/孩子等）
- spice_level: 辣度偏好
- sweet_level: 甜度偏好
- salt_level: 咸淡偏好
- dislikes: 忌口食材数组
- favorite_dishes: 最爱菜品ID数组
- hated_dishes: 讨厌菜品ID数组
```

#### 菜单表 (menus)
```sql
- id: 主键
- name: 菜单名称
- scene: 用餐场景
- date: 用餐日期
- member_ids: 用餐成员ID数组
- dish_ids: 菜品ID数组
- servings: 用餐人数
- note: 备注
- created_at: 创建时间
```

## 六、总结

这个设计把原本的企业级媒体管理系统，改造成温馨的家庭点菜工具。核心思路是：

1. **简化**：去掉复杂的企业功能，保留核心的"记录-管理-使用"流程
2. **温馨**：UI风格从商务蓝改为家庭暖色调
3. **实用**：围绕"今天吃什么"这个真实痛点设计功能
4. **有爱**：通过口味档案、家庭角色等设计，体现对家人的关心
