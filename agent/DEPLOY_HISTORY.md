## 关键决策

### 1. 技术栈选择
- **Next.js**: 服务端渲染、API Routes
- **MySQL**: 关系型数据，适合结构化数据
- **Redis**: 缓存热点数据
- **Tailwind**: 快速样式开发

### 2. 架构设计
- 三个独立系统，统一登录
- 用户数据完全隔离
- 管理员特殊权限

### 3. 部署策略
- 单服务器部署
- Nginx 反向代理
- systemd 服务管理
- Git 版本控制

## 问题与解决

| 问题 | 解决 |
|------|------|
| Redis 缓存不刷新 | FLUSHDB 清除缓存 |
| 前端构建类型错误 | 修复 TypeScript 类型 |
| 端口暴露问题 | 绑定 localhost |
| 上传后列表不显示 | 添加时间戳防缓存 |
| SSH 连接不稳定 | 优化构建流程 |
| 二维码无法扫描 | 使用 QRCode.js 库 |
| 华为微信分享失败 | 使用 Android intent |
